Vibe coding gets the PR merged. Spec-driven development gets the right thing merged.

GitHub open-sourced spec-kit this week—a toolkit for writing product scenarios first, then letting agents implement against them rather than guessing intent from a one-line prompt.

The idea isn't new. It's essentially test-driven development shifted up one level: from code behavior to product behavior. What makes it interesting now is that it pairs well with AI coding agents. A spec gives the agent bounded scope and a clear definition of done. Instead of 'build me a login flow,' you write 'a user with a valid email logs in and sees their dashboard; an invalid email shows an inline error.' The agent has something to verify against, not just something to interpret.

The practical value for solution delivery work: it forces the 'what does success actually look like' conversation before any code is written—which is where most project risk really lives, regardless of whether agents are involved.
