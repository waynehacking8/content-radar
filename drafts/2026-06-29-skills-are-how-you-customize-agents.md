Matt Pocock published his personal .claude directory — the actual agent skills he uses daily for real engineering work, not demo prompts.

What's interesting isn't any single skill, it's the philosophy. Most agentic workflows hand the model a goal and hope for the best. Skills are different: they're reusable, composable instructions that encode how you want the agent to handle specific situations — code review, refactoring, test generation. More like functions than prompts.

This connects to something I keep coming back to in agentic system design: the leverage isn't in the model, it's in the scaffolding around it. A well-crafted set of skills makes the same underlying model dramatically more reliable for your specific workflow and codebase conventions.

Open-sourcing your agent configuration is a new kind of knowledge-sharing. The people who will get the most out of AI-assisted engineering aren't the ones with access to the best model — they're the ones who've put the most thought into how they configure it.
