Most of the AI coding agent conversation is about writing code. Alibaba's Page Agent does something different — it lives inside a webpage and lets you control any web UI with natural language, no API required. Click here, fill this form, extract that table, described in plain English instead of scripted with Selenium.

That's a bigger deal than it sounds for anyone doing forward-deployed or integration work. A huge amount of real enterprise software has no usable API — just a legacy web app that some other team's engineers left behind, or a vendor tool that only exposes a UI. Traditional automation against that surface is brittle: one CSS class changes and your script breaks. A natural-language agent reasoning about "the page," not "the selector," tolerates that kind of drift far better.

It's not a replacement for a real API integration when one exists — direct integration will always be faster and more reliable. But for the long tail of systems that will never get one, an agent that operates the UI the way a human would is a genuinely practical bridge, not just a demo.
