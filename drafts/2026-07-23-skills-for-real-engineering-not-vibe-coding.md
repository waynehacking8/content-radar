Matt Pocock open-sourced the actual Claude Code skills he uses day to day, and the framing is worth more than the skills themselves: "real engineering, not vibe coding."

That distinction matters. Vibe coding is prompt, look at the output, prompt again, hope it converges. It works for a demo. It falls apart the moment you need something maintainable, reviewable, or reproducible by someone other than you. Skills are the opposite instinct: encode the process, not just the request. A skill for writing a migration, reviewing a diff, or scaffolding a feature carries the judgment calls a senior engineer would make automatically, so the agent doesn't have to rediscover them every session.

The pattern generalizes past any one tool. If you find yourself giving an agent the same corrective instructions repeatedly, that's a signal to write it down as a reusable skill instead of a prompt you retype. It's the difference between an agent that helps once and one that helps consistently.
