"Vibe coding" and "real engineering" are in tension, and Matt Pocock's skills repo draws that line explicitly.

The repo is a set of Claude Code skills for actual software development — not demos. The README opens with "not vibe coding" and that framing is doing real work.

Here's the distinction that matters: vibe coding gets you to a working prototype fast. Real engineering means the prototype needs to survive code review, tests, CI, and a production deployment. The gap between those two things is where agent skills become interesting as an abstraction.

A skill is essentially a reusable, named workflow — structured inputs, a consistent execution pattern, a predictable output. When you share a skills file, you're sharing a repeatable process, not just a clever prompt. That's the same idea as a runbook or a deployment checklist, except the executor is the model.

If you're using AI coding assistants in professional contexts, formalizing your skills is a real leverage multiplier. The tool is less important than the habit of making your workflows explicit and portable.
