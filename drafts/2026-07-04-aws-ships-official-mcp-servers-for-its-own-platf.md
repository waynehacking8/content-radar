When the cloud provider ships official MCP servers for its own platform, that's not a side project.

AWS just released the Agent Toolkit for AWS — official MCP servers, skills, and plugins that let AI coding agents build, deploy, and manage on AWS with guardrails included. The word "official" matters here. This isn't a community wrapper; it's AWS saying the agentic workflow is now a first-class path for infrastructure work.

For practitioners, this closes a gap that's been frustrating: coding agents are good at writing Terraform or CDK, but they had no authoritative way to verify what was actually deployed or query live infrastructure state. MCP servers with AWS's backing mean agents can do that with access controls the platform team has already reviewed.

If you're working on Solutions Architect or forward-deployed problems, this is worth tracking. The question shifts from "can the agent write infra code" to "can the agent manage live infrastructure responsibly." AWS is betting on yes.
