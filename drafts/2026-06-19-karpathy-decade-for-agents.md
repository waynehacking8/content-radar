Andrej Karpathy says it'll take a decade to work through the problems with AI agents. I think he's being optimistic.

His point isn't that agents are bad — it's that we're still in the bicycle phase. You can explain how a bicycle works without ever riding one, but you only understand what's missing when you're actually on it.

The agents breaking in production today aren't breaking because the models are weak. They're breaking because the scaffolding is immature. We still don't have solid primitives for long-running agent state, reliable error recovery that doesn't spiral, meaningful observability into what an agent actually decided and why, or cost controls that work under real load.

Foundational infrastructure always takes longer than the demos suggest. The model capabilities are moving fast. The platform layer that makes those capabilities production-safe is lagging.

If you're evaluating agents for real deployments, calibrate accordingly. The demos are real. The production readiness isn't there yet.
