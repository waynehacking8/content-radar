# AI Radar — 2026-07-02

## Themes

### "Software factories" and Forward Deployed Engineers go mainstream

At AI Engineer World's Fair, leaders from Cursor, Warp, and Sierra converged on the same framing: enterprises are moving from ad-hoc AI coding assistants to automated 'software factories' staffed by Forward Deployed Engineers who configure agent loops for customers. Warp's Zach Lloyd argues every major software project will soon run this way, Cursor's Pauline Brunet describes her FDE team doing literal factory setup work inside client orgs, and Sierra's Natalie Meurer sees product engineering and forward-deployment roles converging. This is less a product launch than a signal that the industry is standardizing how agentic coding gets operationalized at scale.

Sources: <https://cursor.com> · <https://warp.dev> · <https://sierra.ai>

### Agent orchestration frameworks keep multiplying

A cluster of new repos targets the same problem — coordinating multiple AI agents — from different angles: msitarzewski/agency-agents packages a roster of specialized persona-agents, langchain-ai/deepagents offers an opinionated 'batteries-included' agent harness, google/agents-cli brings agent tooling to Gemini Enterprise, herdr is a terminal multiplexer for running many coding agents at once, and council-of-high-intelligence runs 18 LLM personas through structured multi-round deliberation. The pattern mirrors the FDE/software-factory theme above: the ecosystem is converging on orchestration-layer tooling as the next infrastructure layer above single-model coding assistants.

Sources: <https://github.com/msitarzewski/agency-agents> · <https://github.com/langchain-ai/deepagents> · <https://github.com/google/agents-cli> · <https://github.com/ogulcancelik/herdr> · <https://github.com/0xNyk/council-of-high-intelligence>

### Agentic offensive-security tools proliferate

Autonomous pentesting is becoming a crowded category: usestrix/strix (1,211 stars) and Unclecheng-li/VulnClaw both wrap LLM agents around the full recon-to-report pentest workflow with GitHub/CI integration, while older recon utilities like Mebus/cupp (password profiling) and soxoj/maigret (username OSINT dossiers) are seeing renewed traction, likely as building blocks agents now call into. Worth flagging for defenders: these lower the skill floor for both red-team engagements and less-authorized use.

Sources: <https://github.com/usestrix/strix> · <https://github.com/Unclecheng-li/VulnClaw> · <https://github.com/Mebus/cupp> · <https://github.com/soxoj/maigret>

### Claude Code's surface area keeps expanding beyond coding

Several launches use Claude Code as a general agent runtime rather than just a coding tool: browser-use/video-use turns it into a video editor (drop footage in, chat, get a final cut), virgiliojr94/book-to-skill converts technical books into reusable Claude Code skills, and diegosouzapw/OmniRoute is a free multi-provider AI gateway (231+ providers) built to plug into Claude Code, Codex, Cursor, and Copilot alike. Together they point to Claude Code's skill/agent architecture becoming a general integration point for non-coding workflows.

Sources: <https://github.com/browser-use/video-use> · <https://github.com/virgiliojr94/book-to-skill> · <https://github.com/diegosouzapw/OmniRoute>

### Sonnet 5 ships, Fable 5 next

Anthropic released Claude Sonnet 5, with AINews framing it as reopening a fast-moving model-release cadence ('Everything is open again') and previewing Fable 5 for the following day. Coming alongside Ahmad Osman's AIEWF talk on local AI catching up on laptops and phones, the frontier/local dichotomy is a live tension right now rather than a settled story.

Sources: <https://substack.com>
