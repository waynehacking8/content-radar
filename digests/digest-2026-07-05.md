# AI Radar — 2026-07-05

> The agent tooling ecosystem is consolidating around 'skills' as the standard extensibility layer, while a wave of new repos tackles orchestrating and securing the growing swarm of coding agents.

*Checked arxiv(30), github(25), gmail(10) — 65 items.*

## Themes

### Agent Skills becomes the de facto standard

A formal Agent Skills spec (agentskills/agentskills) is gaining traction alongside a flood of curated skill packs — mattpocock's real-engineer skills, dotnet's official .NET agent skills, a 337-skill mega-collection from alirezarezvani, and the long-running awesome-claude-code list. The pattern is clear: teams are converging on lightweight, shareable capability packages rather than one-off prompts, with even joke entries like the 'caveman' token-reduction skill (65% fewer output tokens by talking in caveman-speak) riding the same packaging convention.

Sources: <https://github.com/agentskills/agentskills> · <https://github.com/mattpocock/skills> · <https://github.com/dotnet/skills> · <https://github.com/alirezarezvani/claude-skills> · <https://github.com/hesreallyhim/awesome-claude-code> · <https://github.com/JuliusBrussee/caveman>

### Orchestrating a fleet of coding agents

As developers run multiple AI coding agents at once, tooling is emerging to manage them: herdr is a terminal-based agent multiplexer that shows which agents are blocked, working, or done, while OpenAI's codex-plugin-cc lets Claude Code users delegate reviews to Codex directly. This mirrors the broader AIEWF discussion (via AINews) of 'the great loops debate' — how much autonomy to give agents and how humans stay in the loop steering them.

Sources: <https://github.com/ogulcancelik/herdr> · <https://github.com/openai/codex-plugin-cc>

### Agents get hands: browser, page, and Unity control

Multiple releases this week give agents direct control over live interfaces rather than just generating text: Alibaba's page-agent is an in-page JS GUI agent that manipulates web UIs via natural language, ChromeDevTools/chrome-devtools-mcp exposes a live Chrome browser to coding agents like Claude and Cursor, and CoplayDev/unity-mcp bridges LLMs into the Unity Editor for scene and asset manipulation. Together they point to agents increasingly acting as operators of existing software surfaces instead of only producing code.

Sources: <https://github.com/alibaba/page-agent> · <https://github.com/ChromeDevTools/chrome-devtools-mcp> · <https://github.com/CoplayDev/unity-mcp>

### Offense and exposure: AI pentesting and leaked system prompts

usestrix/strix, the top-scoring item today, packages autonomous 'AI hackers' to find and fix app vulnerabilities — a sign that offensive security tooling is now a first-class AI-agent use case. Meanwhile asgeirtj/system_prompts_leaks continues to collect extracted system prompts from Anthropic, OpenAI, Google, and xAI's production models, and an arXiv paper on 'Distributed Attacks in Persistent-State AI Control' formalizes the risk that a compromised coding agent can plant attacks that persist across sessions in a codebase.

Sources: <https://github.com/usestrix/strix> · <https://github.com/asgeirtj/system_prompts_leaks> · <https://arxiv.org/abs/2607.02514v1>

### AIEWF aftermath: agents as a new software category

Several AINews dispatches recap the AI Engineer World's Fair, where Vercel's Andrew Qu argued agents are 'a new kind of software' built around skills and sandboxes, Adobe demoed self-assembling 'agentic sites' that generate pages per visitor intent, and Paul Bakaus pushed back on one-shot AI design in favor of 'skill engineering' with humans steering loops. The throughline across these talks is the same tension showing up in the GitHub signal: how much structure (skills, sandboxes, multiplexers) agents need before autonomy becomes trustworthy.

Sources: <https://arxiv.org/abs/2607.02436v1>
