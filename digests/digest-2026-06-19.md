# AI Radar — 2026-06-19

> Agent skill ecosystems and code-intelligence tooling converge: persistent knowledge graphs promise 99% token reduction while OpenAI, Anthropic, and independents all ship composable skills catalogs the same week.

## Themes

### Code Intelligence as Context Compression

The top-scoring item, codebase-memory-mcp, indexes codebases into a persistent knowledge graph with sub-millisecond queries and claims 99% token reduction — directly attacking the context-window tax that makes large-repo AI coding expensive. Two smaller projects (cocoindex-code, code-review-graph) take the same AST-based angle at lower scores, and an arxiv paper ('Probe-and-Refine Tuning') formalizes the underlying problem: coding agents need high-level operational knowledge about repos that doesn't exist in raw source. The cluster suggests persistent, queryable codebase indexes are becoming a standard layer beneath coding agents rather than a nice-to-have.

Sources: <https://github.com/DeusData/codebase-memory-mcp> · <https://github.com/cocoindex-io/cocoindex-code> · <https://github.com/tirth8205/code-review-graph> · <https://arxiv.org/abs/2606.20512v1>

### Agent Skill Ecosystems Reach Critical Mass

OpenAI shipped a Skills Catalog for Codex agents; Anthropic released a financial-services reference agents repo covering investment banking, equity research, and wealth management; obra/superpowers landed as a composable skills methodology for coding agents; and K-Dense-AI's scientific-agent-skills library claims 160K scientist users with 140 ready-to-use skills. All four dropped within the same signal window, signaling a platform shift: agent capability is moving toward modular, reusable skill packs rather than monolithic system prompts, and every major player is racing to define the catalog format.

Sources: <https://github.com/openai/skills> · <https://github.com/anthropics/financial-services> · <https://github.com/obra/superpowers> · <https://github.com/K-Dense-AI/scientific-agent-skills>

### Agentic Dev Platforms and Methodology Harden

Kilo Code (score 1345) positions itself as an open-source, all-in-one agentic engineering platform with multilingual reach, while GitHub's spec-kit (score 764) pushes spec-driven development as an alternative to 'vibe coding' — write scenarios and outcomes first, let agents fill in the implementation. Astro's Flue adds a TypeScript-native sandbox harness for composing agent workflows. Together these represent infrastructure maturing past 'AI as autocomplete' toward structured, auditable agent pipelines with defined inputs and expected outputs.

Sources: <https://github.com/Kilo-Org/kilocode> · <https://github.com/github/spec-kit> · <https://github.com/withastro/flue>

### New Model Releases: Time Series and Video

Google Research's TimesFM (score 844) drew the most attention in the model space — a pretrained time-series foundation model for forecasting that sidesteps task-specific training. Lightricks shipped LTX-2, a DiT-based audio-video foundation model that generates synchronized audio and video in one pass, and Roboflow's RF-DETR earned an ICLR 2026 acceptance for real-time object detection and segmentation SOTA on COCO. GLM-5.2 (score 202) claims top position on frontend coding benchmarks as a strong open-weight competitor. No single release dominates, but the breadth signals continued diversification beyond text LLMs.

Sources: <https://github.com/google-research/timesfm> · <https://github.com/Lightricks/LTX-2> · <https://github.com/roboflow/rf-detr> · <https://github.com/zai-org/GLM-5>

### Agent Security and Reliability Get Formal Treatment

Three arxiv papers address different failure modes of deployed agents: 'Sovereign Execution Brokers' argues that production mutation authority must not live inside non-deterministic reasoning processes and proposes certificate-bound authority enforcement; 'Contagion Networks' shows how evaluator bias propagates through multi-agent LLM systems with measurable network effects; and 'Efficient and Sound Probabilistic Verification for AI Agents' applies Datalog-based runtime monitoring to enforce formal policies. The common thread is a shift from 'trust the model' to 'verify the agent at the infrastructure layer' — an engineering discipline emerging alongside agent deployment at scale.

Sources: <https://arxiv.org/abs/2606.20520v1> · <https://arxiv.org/abs/2606.20493v1> · <https://arxiv.org/abs/2606.20510v1>
