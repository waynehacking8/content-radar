# AI Radar — 2026-07-09

> The 'skill' has become the default unit of agent capability — a wave of skill libraries, optimizers, and security scanners shipped in parallel — while Claude Code itself keeps getting repurposed as a general automation substrate for non-coding tasks like job hunting and video analysis.

## Themes

### Skills become the primary abstraction for coding agents

This cycle formalizes 'skills' as the reusable unit of agent capability rather than one-off prompts: addyosmani/agent-skills packages senior-engineer workflows and quality gates, obra/superpowers builds a full development methodology on composable skills, and alirezarezvani/claude-skills ships 345 skills/agents/commands across eight agent platforms. Microsoft's SkillOpt and the SkillCenter paper push further, trying to make skill libraries self-optimizing and source-grounded instead of hand-curated. NVIDIA shipping SkillSpector — a security scanner for agent skills — in the same window signals the ecosystem is now mature (and risky) enough that skill supply-chain safety is a real concern, not a hypothetical.

Sources: <https://github.com/addyosmani/agent-skills> · <https://github.com/obra/superpowers> · <https://github.com/alirezarezvani/claude-skills> · <https://github.com/microsoft/SkillOpt> · <https://github.com/NVIDIA/SkillSpector> · <https://arxiv.org/abs/2607.07676v1>

### Claude Code as a general automation substrate, not just a coding tool

Several high-engagement projects use Claude Code to orchestrate tasks well outside software engineering. MadsLorentzen's ai-job-search (the top item today at 5k+ stars) automates job hunting end-to-end — evaluating postings, tailoring CVs, prepping interviews — while bradautomates/claude-video gives Claude a /watch command to download, transcribe, and reason over arbitrary video. iOfficeAI's OfficeCLI lets agents edit Word/Excel/PowerPoint without Office installed, and Graphify turns any folder of code, docs, or media into a queryable knowledge graph for Claude Code, Codex, Cursor, and others. The common thread: agent harnesses are becoming the integration layer of choice for everyday workflows, not just dev tasks.

Sources: <https://github.com/MadsLorentzen/ai-job-search> · <https://github.com/bradautomates/claude-video> · <https://github.com/iOfficeAI/OfficeCLI> · <https://github.com/Graphify-Labs/graphify>

### System prompt leaks renew scrutiny of model transparency

asgeirtj/system_prompts_leaks — an aggregation of extracted system prompts from Claude, ChatGPT, Gemini, Grok, Cursor, and Copilot — pulled over 1,200 points today, underscoring persistent community appetite for seeing what vendors actually instruct their models to do behind the scenes. The interest level suggests this remains a live transparency/trust fault line even as these assistants get embedded deeper into daily workflows.

Sources: <https://github.com/asgeirtj/system_prompts_leaks>

### Agent infrastructure hardens: sandboxes, memory, SRE tooling

Behind the flashy agent demos, infra providers are racing to productize the plumbing agents need to run safely and persistently. Tencent shipped both CubeSandbox (concurrent, secure sandboxing for agent code execution) and TencentDB-Agent-Memory (fully local, zero-API long-term memory via a four-tier pipeline), while Tracer-Cloud's opensre offers an open toolkit for building and evaluating AI SRE agents. Together they point to 2026's agent tooling gap shifting from 'can it reason' to 'can it run reliably and remember things without leaking data.'

Sources: <https://github.com/TencentCloud/CubeSandbox> · <https://github.com/TencentCloud/TencentDB-Agent-Memory> · <https://github.com/Tracer-Cloud/opensre>

### Local-first, cheap AI keeps shrinking further

A cluster of releases pushes AI capability down to commodity hardware: Kyutai's pocket-tts runs text-to-speech entirely on CPU, Alibaba's zvec is an in-process vector database with no server dependency, and maziyarpanahi/openmed delivers on-device clinical NER and HIPAA de-identification with zero cloud calls. At the extreme end, jingyaogong/minimind demonstrates training a 64M-parameter LLM from scratch for roughly $3 and two hours of compute — a reminder that the barrier to building (not just running) small models keeps dropping.

Sources: <https://github.com/kyutai-labs/pocket-tts> · <https://github.com/alibaba/zvec> · <https://github.com/maziyarpanahi/openmed> · <https://github.com/jingyaogong/minimind>
