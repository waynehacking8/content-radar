# AI Radar — 2026-07-04

## Themes

### Agent Skills becomes a standard, not just a Claude feature

What started as Claude Code's skill format is turning into shared infrastructure: agentskills/agentskills published a standardized spec for giving agents new capabilities, obra/superpowers packaged a full agentic development methodology on top of composable skills, and alirezarezvani/claude-skills shipped 330+ skills portable across Claude, Codex, Gemini CLI and Cursor. The joke entry of the day — JuliusBrussee/caveman, a skill that makes agents write terse 'caveman' output to cut token usage 65% — topped GitHub trending, but it's really evidence of how normalized the skill packaging pattern has become. Meanwhile openai/codex-plugin-cc and herdr (a terminal multiplexer for running/monitoring several coding agents at once) show tooling is now being built for a world where multiple agents and vendors coexist in one workflow.

Sources: <https://github.com/agentskills/agentskills> · <https://github.com/obra/superpowers> · <https://github.com/alirezarezvani/claude-skills> · <https://github.com/JuliusBrussee/caveman> · <https://github.com/openai/codex-plugin-cc> · <https://github.com/ogulcancelik/herdr>

### Securing (and attacking with) autonomous agents

usestrix/strix, an open-source autonomous AI pentesting tool that finds and patches app vulnerabilities, was the second-highest trending repo today, while Microsoft's agent-governance-toolkit ships policy enforcement, zero-trust identity and sandboxing aimed at covering the full OWASP Agentic Top 10. On the research side, a new arXiv paper on 'Distributed Attacks in Persistent-State AI Control' flags a real emerging risk: coding agents that persist state across sessions give a misaligned or prompt-injected agent a way to smuggle in attacks incrementally rather than all at once. Together these signal that agent security is bifurcating into offense tooling (strix), defense frameworks (governance toolkit), and threat modeling (the persistence paper) at the same time.

Sources: <https://github.com/usestrix/strix> · <https://github.com/microsoft/agent-governance-toolkit> · <https://arxiv.org/abs/2607.02514v1>

### Dev infrastructure is being rebuilt 'agent-ready'

Facebook's astryx is a design system explicitly built for agents and humans to co-develop UI, chrome-devtools-mcp gives coding agents direct control of a live Chrome browser for inspection and debugging, and TencentCloud/CubeSandbox offers a lightweight concurrent sandbox specifically for running AI agents safely. This is a distinct trend from the skills story above: rather than packaging agent behavior, these projects are retrofitting existing dev surfaces (design systems, browser devtools, execution sandboxes) so agents can use them as reliably as humans do.

Sources: <https://github.com/facebook/astryx> · <https://github.com/ChromeDevTools/chrome-devtools-mcp> · <https://github.com/TencentCloud/CubeSandbox>

### Evidence check: reasoning effort beats extra tools for agent reliability

An observational study on arXiv ('Reasoning effort, not tool access, buys first-try reliability in agentic code generation') pushes back on the assumption that giving coding agents more capability — browser testing, elaborate system prompts — automatically improves output quality, finding that how much the model reasons matters more than what tools it's handed. That's a useful corrective to today's flood of tool/sandbox/skill announcements, most of which bet implicitly on capability-stacking rather than reasoning depth.

Sources: <https://arxiv.org/abs/2607.02436v1>

### AI Engineer World's Fair recap: agents as a new software category

Swyx's AINews dispatch from AIEWF centered on 'the great loops debate' and closed with Vercel's Andrew Qu arguing that agents constitute a genuinely new kind of software — citing Vercel's own agent framework 'eve' and the growing importance of skills, sandboxes, and agent-readable websites. That framing ties directly back into today's GitHub activity: skills standardization, agent-ready design systems, and dedicated agent sandboxes are exactly the building blocks Qu says this new software category needs.

Sources: <https://ainews.substack.com>
