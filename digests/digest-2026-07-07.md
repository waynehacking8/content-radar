# AI Radar — 2026-07-07

> The day's signal is dominated by the Claude Code skills/plugins gold rush, alongside a mass leak of frontier model system prompts and a live debate over how agent 'loops' should be architected for long-horizon work.

## Themes

### Claude Code's skills/plugins ecosystem is exploding

The top of the board is almost entirely coding-agent tooling built on or around Claude Code: taste-skill (anti-slop generation), addyosmani's production-grade engineering skills, a Codex-from-Claude-Code plugin, herdr (a terminal multiplexer for running several agents at once), 345-skill packs from alirezarezvani, and CodexBar for tracking usage limits. The pattern is a rapid shift from 'agents that write code' to 'meta-tooling for managing fleets of agents' — orchestration, usage monitoring, and reusable skill libraries are now bigger growth areas than the underlying models themselves.

Sources: <https://github.com/Leonxlnx/taste-skill> · <https://github.com/addyosmani/agent-skills> · <https://github.com/openai/codex-plugin-cc> · <https://github.com/ogulcancelik/herdr> · <https://github.com/alirezarezvani/claude-skills> · <https://github.com/steipete/CodexBar>

### Frontier system prompts leaked en masse

asgeirtj/system_prompts_leaks (1378 pts) collects extracted system prompts spanning Anthropic (Fable 5, Opus 4.8, Claude Code), OpenAI (ChatGPT 5.5, Codex), Google (Gemini 3.5/3.1, Antigravity), and xAI/Grok, plus IDE tools like Cursor and Copilot. Wide interest in this suggests prompt engineering practices at the big labs are becoming a de facto shared reference point, and that extraction/leak techniques are now good enough to reliably surface these across nearly every major assistant at once.

Sources: <https://github.com/asgeirtj/system_prompts_leaks>

### The 'agent loops' debate goes from conference stage to research

AINews' AIEWF recap frames the World's Fair closing out on a fight over how agent control loops should be designed — timely given same-day arXiv work like CompactionRL (RL with context compaction for long-horizon agents) and MetaSkill-Evolve (recursive self-improvement via meta-skill evolution), both directly attacking the finite-context, single-pass loop model. The throughline: as agents run longer tasks, the naive 'one big context, one pass' loop is being replaced by compaction, skill-reuse, and self-modifying strategies — an actual architectural shift, not just a vibes debate.

Sources: <https://arxiv.org/abs/2607.05378v1> · <https://arxiv.org/abs/2607.05297v1>

### Local-first infra for building AI apps

Meetily (2494 pts, the top item overall) pitches a fully local, Rust-based meeting assistant using 4x-faster Parakeet/Whisper transcription plus Ollama summarization — no cloud dependency. Alongside it, Firecrawl continues as the default 'give an LLM the web' scraping API, and Alibaba's zvec adds a lightweight in-process vector database. Together these point to steady demand for privacy-preserving, self-hostable building blocks rather than API-only SaaS wrappers.

Sources: <https://github.com/Zackriya-Solutions/meetily> · <https://github.com/firecrawl/firecrawl> · <https://github.com/alibaba/zvec>

### Agentic frameworks push further into finance

TradingAgents (322 pts, now at v0.3.1) is a multi-agent LLM framework for financial trading, sitting alongside Microsoft's Qlib quant platform and Anthropic's own new financial-services reference repo (skills/connectors for investment banking, equity research, and wealth management). Finance is emerging as one of the few verticals where multiple independent players are simultaneously shipping purpose-built agent stacks rather than generic assistants.

Sources: <https://github.com/TauricResearch/TradingAgents> · <https://github.com/microsoft/qlib> · <https://github.com/anthropics/financial-services>
